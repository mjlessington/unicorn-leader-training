# Module 4: Courageous Leadership in Tech

**Duration:** 90 minutes | **Format:** Case Studies + Role Play + Strategy Design

---

## Learning Objectives

By the end of this module, participants will be able to:

1. Apply Brown's framework for brave conversations to common IT leadership scenarios
2. Lead organizational change using vulnerability and unicorn habits together
3. Create a culture of innovation that embraces productive failure
4. Navigate the specific courage challenges unique to technology leadership

## Courage Is Not Comfort

Brown's central thesis in *Dare to Lead* is that you cannot get to courage without walking through vulnerability. There is no shortcut, no hack, no process that removes the emotional exposure.

For IT leaders — trained in systems, processes, and optimization — this is a challenging truth. We want to engineer courage. We want a framework that makes hard conversations easy. We want a process that removes the risk from innovation.

None of that exists. What does exist: **a set of skills that help you show up in the hard moments with more clarity, more connection, and more effectiveness.**

This module focuses on three domains where IT leaders most need courage:
1. **Brave conversations** — the discussions you've been avoiding
2. **Leading through change** — transformation that doesn't destroy trust
3. **Innovation and failure** — creating space for the risk that progress requires
